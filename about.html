Introduction

For my website, I created a marketing website for monopoly tokens. I added a 3D model of three of the tokens with picture labels and sound effects of the sounds these objects would typically make. I chose monopoly as it’s a well known game with lots of history and recognisable pieces.

3D Models

For this website, I created a 3D model of a boot, cat and ship in blender. I used features like extrude, shade smooth, subdivision surface and bevel. I formatted the exports in .glb as it works the best with three.js, sometimes referred to as “the JPEG of 3D”.

Usability testing

I structured the page so that every page has the Monopoly logo, nav bar, monopoly man and the same colour scheme. The navbar has home, tokens (which drops down for the three tokens: boot, cat and ship), and the about page. As the actual monopoly tokens are silver (all the same colour), I decided to make the website itself more colourful to go with the theme and to make it more complex and aesthetically pleasing. I set up the three.js scene using javascript within the individual html files, namely lights, camera and rendering.

Integration

I loaded the .glb files into three.js by using TGLFLoader which is a three.js plug-in designed for .glb files. The pages linked together as each token has their own html file and they all come under the tokens dropdown in the navigation bar. Also, if you click on one of the token images it will take you to said html page. All of the pages have the same css so the design is the same. The CSS and js work together as css controls the UI - so whatever the user sees in terms of colours, button shapes, etc. whereas js handles the functionality e.g. if you click a button, an object moves.

Functional testing

Each button has a different animation e.g. toggle wireframe strips the model of the solid part and only shows the wireframe, Js was used for buttons, rotate makes the model spin in an anti-clockwise direction on the y-axis, stomp/meow/skim makes the model move up and down fast as if it’s stomping compared to walk/swim being slow using a sine wave, the light gets dimmer and brighter and orbitControls lets you drag and move the object in whichever angle you want and scroll to zoom. There is one sound effect for every token: the boot makes a stomp noise when it stomps, the cat runs making a meow noise when you click meow and the ship makes a fast noise when you skim.
